# Output Specification

## Overview
The application produces a single XML route file per processing run. No additional logs or reports are generated by default.

## File Naming and Location
- **Location**: The selected `output` folder.
- **Filename**: Use a clear pattern, for example: `<MMSI>_<startYYYYMMDDTHHMMSS>_<endYYYYMMDDTHHMMSS>.xml`

## Content
- Follow the structure in `route_waypoint_template.xml` provided in the repository. Ensure waypoints include time, latitude, longitude and any required attributes per template.
